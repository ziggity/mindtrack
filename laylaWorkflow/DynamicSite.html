Deploy a Dynamic Website on AWS EC2

A Detailed Guide to Hosting Your App & Database on a Single Server

Architecture Overview

This guide will walk you through setting up a traditional, server-based dynamic website. We will use a single AWS EC2 instance (a virtual server) to host both the web application files and the database. This is a powerful and flexible approach that gives you complete control over your server environment.

Step 1: Launch Your EC2 Virtual Server

First, we need to create the virtual server that will run our website.

  1. Navigate to the EC2 service in your AWS Console and click "Launch instance".
  2. Name and tags: Give your server a descriptive name, like `my-dynamic-website-server`.
  3. Application and OS Images (AMI): Choose an Amazon Machine Image. Amazon Linux 2 is a great, free-tier eligible choice.
  4. Instance type: Select t2.micro. This is crucial to stay within the AWS Free Tier for new accounts.
  5. Key pair (login): This is your server's password. Click "Create new key pair", give it a name (e.g., `my-website-key`), and select `.pem` format. The key file will download immediately. Keep this file safe and private! You will not be able to download it again.
  6. Network settings: Click "Edit". In the "Security group" section, create a new one with a descriptive name. Add the following inbound rules:
    • Type `SSH` | Port `22` | Source: `My IP` (This allows you to securely connect to the server from your current IP address).
    • Type `HTTP` | Port `80` | Source: `Anywhere` (This allows visitors to access your website).
    • Type `HTTPS` | Port `443` | Source: `Anywhere` (For future use when you add an SSL certificate).
  7. Review the summary and click "Launch instance".

Step 2: Assign a Static IP Address

A new server's public IP address can change if it's stopped and restarted. An Elastic IP is a static, public IP address you can assign to your instance to prevent this.

  1. In the EC2 console menu, go to "Elastic IPs".
  2. Click "Allocate Elastic IP address" and then "Allocate".
  3. Select the newly created IP, click "Actions", and choose "Associate Elastic IP address".
  4. In the dialog, select your running EC2 instance and click "Associate". Your server now has a permanent public IP.

Step 3: Install the Web Stack (LAMP)

Now, we'll connect to the server and install the necessary software: Linux (already running), Apache (web server), MariaDB (database), and PHP (programming language).

Connect via SSH

Open a terminal (on macOS/Linux) or an SSH client like PuTTY (on Windows). Use the following command, replacing the placeholders:

# Make your key file read-only first
chmod 400 /path/to/my-website-key.pem

# Connect to the server
ssh -i /path/to/my-website-key.pem ec2-user@YOUR_ELASTIC_IP

Run Installation Commands

Once connected, run these commands one by one:

# 1. Update all installed packages to their latest versions
sudo yum update -y

# 2. Install Apache, MariaDB, and PHP with the MySQL connector
sudo yum install -y httpd mariadb-server php php-mysqlnd

# 3. Start the Apache web server now
sudo systemctl start httpd

# 4. Configure Apache to start automatically on server boot
sudo systemctl enable httpd

# 5. Start the MariaDB database server now
sudo systemctl start mariadb

# 6. Configure MariaDB to start automatically on server boot
sudo systemctl enable mariadb

# 7. Run the security script to set a root password and secure the database
sudo mysql_secure_installation

For the `mysql_secure_installation` script, press ENTER for the first prompt (current root password), then 'Y' to set a new root password. For all subsequent questions, it's safe to answer 'Y' (yes).

Step 4: Create Your Database and User

It's best practice to create a dedicated user for your application instead of using the root user.

-- 1. Log in to MariaDB as the root user
sudo mysql -u root -p

-- 2. Create a new database
CREATE DATABASE my_webapp_db;

-- 3. Create a new user that can only be accessed from the server itself ('localhost')
CREATE USER 'my_webapp_user'@'localhost' IDENTIFIED BY 'YourStrongPassword123!';

-- 4. Grant all privileges on your new database to your new user
GRANT ALL PRIVILEGES ON my_webapp_db.* TO 'my_webapp_user'@'localhost';

-- 5. Apply the changes
FLUSH PRIVILEGES;

-- 6. Exit the database client
EXIT;

Step 5: Set Up a Users Table in the Database

Now, let's create a table within our new database to store user information. Log back into MariaDB and run the following command.

-- 1. Log in to MariaDB as the root user
sudo mysql -u root -p

-- 2. Select the database you want to use
USE my_webapp_db;

-- 3. Create the 'users' table
CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    registration_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 4. Exit the database client
EXIT;

Security Best Practice

Notice we named the column `password_hash`. We will never store passwords directly. Instead, we will store a secure, one-way hash of the password. This means even if the database is compromised, the actual passwords are not exposed.

Step 6: Create the User Registration App

We'll create three files in the web root (`/var/www/html`): an index page, an HTML form for registration, and a PHP script to handle the form submission.

1. Create the Index Page (`index.html`)

This will be a simple landing page. Run `sudo nano /var/www/html/index.html` and add this content:

<!DOCTYPE html>
<html>
<head><title>Welcome</title></head>
<body>
  <h1>Welcome to My Dynamic Website</h1>
  <a href="register.html">Register a new user</a>
</body>
</html>

2. Create the Registration Form (`register.html`)

This form will collect the user's desired username and password. Run `sudo nano /var/www/html/register.html` and add this:

<!DOCTYPE html>
<html>
<head><title>User Registration</title></head>
<body>
  <h2>Register</h2>
  <form action="handle_register.php" method="post">
    <label for="username">Username:</label><br>
    <input type="text" id="username" name="username" required><br>
    <label for="password">Password:</label><br>
    <input type="password" id="password" name="password" required><br><br>
    <input type="submit" value="Register">
  </form>
</body>
</html>

3. Create the PHP Handler (`handle_register.php`)

This is the core logic. It takes the form data, hashes the password, and saves it to the database. Run `sudo nano /var/www/html/handle_register.php` and add this script:

<?php
// Database credentials
$servername = "localhost";
$username = "my_webapp_user";
$password = "YourStrongPassword123!";
$dbname = "my_webapp_db";

// Create connection
$conn = new mysqli($servername, $username, $password, $dbname);

// Check connection
if ($conn->connect_error) {
  die("Connection failed: " . $conn->connect_error);
}

// Get username and password from the form
$user = $_POST['username'];
$pass = $_POST['password'];

// --- Security: Hash the password ---
$password_hash = password_hash($pass, PASSWORD_BCRYPT);

// --- Security: Use a prepared statement to prevent SQL injection ---
$stmt = $conn->prepare("INSERT INTO users (username, password_hash) VALUES (?, ?)");
$stmt->bind_param("ss", $user, $password_hash);

// Execute the statement and provide feedback
if ($stmt->execute()) {
  echo "New record created successfully. You can now log in.";
} else {
  echo "Error: " . $stmt->error;
}

$stmt->close();
$conn->close();
?>

Step 7: Point Your Domain to the Server

The final step is to tell your domain name where to find your server.

  1. Navigate to Route 53 in the AWS Console and select your Hosted Zone.
  2. Click "Create record".
  3. For the root domain (`yourdomain.com`): Leave the "Record name" blank, select `A` as the record type, and paste your server's Elastic IP address into the "Value" field. Click "Create records".
  4. For the `www` subdomain: Click "Create record" again. Enter `www` for the "Record name", select `A` as the type, and paste the same Elastic IP address into the "Value" field. Click "Create records".

Step 8: Test and Troubleshoot

Wait a few minutes for the DNS changes to propagate, then open a web browser and navigate to `http://yourdomain.com`. You should see the welcome page with a link to register. Try creating a new user.

Common Issues:

  • Site doesn't load (Connection Timed Out): Double-check that your EC2 Security Group allows HTTP (Port 80) traffic from "Anywhere". Also, ensure your domain's `A` record points to the correct Elastic IP.
  • Registration form gives an error: This often means your PHP script can't connect to the database. Double-check the credentials in `handle_register.php` to ensure they match what you created in Step 4. You can also check the Apache error log for more details: `sudo tail -f /var/log/httpd/error_log`.
  • "Duplicate entry" error on registration: This is good! It means your `UNIQUE` constraint on the `username` column is working correctly. Try registering with a different username.
